# nopCommerce Demo Website - Manual Testing Project

A structured self-practice manual testing project for the nopCommerce demo website, created to simulate a real QA workflow and demonstrate practical testing skills for a Manual Tester role.

## Project Information
- **Application Under Test:** https://demo.nopcommerce.com/
- **Domain:** E-commerce
- **Testing Type:** Manual Testing
- **Project Type:** Personal QA Practice Project

## Scope Covered
The following core user flows were analyzed and tested:
- User Registration
- Login / Logout
- Product Search
- Product Detail Page
- Shopping Cart
- Wishlist
- Checkout Flow
- Basic UI / Validation / Negative Scenarios

## Testing Types Included
- Functional Testing
- UI Testing
- Validation Testing
- Negative Testing
- Smoke Testing
- Regression Testing
- Exploratory Testing

## Deliverables
- Test Cases (60 cases)
- Bug Report (Sample defects)
- Smoke Test Checklist
- Regression Test Checklist
- Test Plan
- Test Summary Report
- RTM (Requirement Traceability Matrix)

## Tools Used
- Microsoft Excel / Google Sheets
- Microsoft Word / Google Docs
- Chrome DevTools
- GitHub / Google Drive

## Project Outcome
- Designed and executed 60+ manual test cases
- Built smoke and regression checklists for critical flows
- Logged sample defects using a structured bug reporting format
- Prepared QA artifacts similar to real project documentation
- Improved understanding of end-to-end e-commerce business flows

## Notes
This is a self-learning QA project created to practice end-to-end manual testing on a realistic e-commerce workflow.
